# BlockPilot Brand Kit

BlockPilot is an AI-native builder for on-chain products. The brand should feel precise, fast, trustworthy, and slightly futuristic without looking like a trading dashboard.

## Positioning

**One-liner:** Prompt, generate, and deploy blockchain projects with AI.

**Short description:** BlockPilot turns natural-language ideas into deploy-ready Web3 projects: smart contracts, frontend files, ZIP export, and MetaMask deployment.

**Voice:** Clear, founder-friendly, practical, and confident. Avoid heavy crypto slang unless speaking to developers.

## Logo

Primary logo assets live in `public/`:

- `logo.svg` — app/logo mark
- `favicon.svg` — browser icon
- `og.svg` — social preview

The mark combines a block/hex shape with a pilot-forward `P`, signalling guided blockchain creation.

## Typography

Use the system stack for speed and crisp rendering:

`system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`

- Hero: bold, direct, sentence case where possible
- Product UI: compact labels, short verbs, concrete nouns
- Docs/copy: practical and builder-oriented

## Messaging Pillars

1. **Prompt-first:** users start with a natural language idea.
2. **Deploy-ready:** output should move toward a real launch, not just code snippets.
3. **Chain-aware:** chain selection, explorer links, wallet flow, and network switching matter.
4. **Founder-friendly:** non-dev users should understand what is happening.
5. **Developer-respectful:** code should be exportable, inspectable, and editable.

## Example Copy

- "Build on-chain products with AI."
- "Prompt → generate → deploy."
- "Turn ideas into deploy-ready Web3 projects."
- "Generate contracts, frontend files, and deployment flow in minutes."

## Avoid

- Overpromising audits or mainnet safety
- Saying generated contracts are production-safe without review
- Meme-heavy crypto language
- One-color purple-only visuals
